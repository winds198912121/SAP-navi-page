## 1. 产品概述

### 1.1 产品定位

SAP パンダ先生 是一个面向日本企业 SAP 学习者和从业者的专业知识培训与求职平台。以治愈系卡通熊猫形象"パンダ先生"为品牌核心，将枯燥难懂的 SAP 知识转化为易于理解和吸收的学习内容，降低 SAP 技术的学习门槛。

### 1.2 产品愿景

让每一位 SAP 学习者和从业者都能通过パンダ先生的引导，轻松掌握 SAP 知识，提升职业技能，获得更好的职业发展机会。

### 1.3 产品口号

> **「SAPの世界を、もっと身近に。」**（让 SAP 的世界，更加贴近你。）

### 1.4 核心价值

| 价值主张 | 说明 |
|---------|------|
| **降低门槛** | 用熊猫先生 vs 学生的对话形式讲解枯燥的 SAP 概念 |
| **模块齐全** | 覆盖 9 大 SAP 模块（FI/CO/MM/SD/PP/HR/ABAP/Basis/S/4） |
| **学习路径** | 按角色和能力水平提供结构化学习路线 |
| **学以致用** | 学习 + 求职一体化，知识变现的闭环 |
| **治愈系体验** | 温暖、友善的视觉设计，减轻学习焦虑 |

## 3. 用户角色定义

### 3.1 角色矩阵

| 角色 | 权限级别 | 主要功能 | 系统访问 |
|-----|---------|---------|---------|
| **访客 (未登录)** | 只读 | 浏览文章、观看视频、查看案件 | 前台全部公开内容 |
| **注册用户** | 基础 | 学习记录、收藏、答题、评论 | 前台 + 个人中心 |
| **付费会员** | 高级 | 学习路径完整访问、AI 问答、证书 | 前台 + 会员专属内容 |
| **内容作者** | 编辑 | 发布/编辑文章、管理课程 | WordPress 后台 |
| **管理员** | 超级管理 | 全部后台管理功能 | WordPress 后台 + 管理系统 |

### 3.2 角色详细说明

#### 3.2.1 访客 (Visitor)
- 可以浏览所有公开文章和视频
- 可以查看案件列表（不能投递）
- 可以参与每日一题（不记录成绩）
- 可以订阅 Newsletter

#### 3.2.2 注册用户 (User)
- 具备访客所有权限
- 学习进度追踪
- 收藏文章
- 答题记录和正解率
- 评论和互动
- 案件投递功能
- 积分累积

#### 3.2.3 付费会员 (Premium)
- 具备注册用户所有权限
- 完整学习路径访问
- AI 助教问答
- PDF 修了证书
- 无广告体验
- 优先案件推荐

#### 3.2.4 内容作者 (Author)
- 具备付费会员所有权限
- 发布/编辑/删除文章
- 管理 quizzes
- 查看内容统计

#### 3.2.5 管理员 (Admin)
- 全部系统管理权限
- 用户管理
- 内容管理
- 案件管理
- 系统配置
- 数据分析

## 4. 用户旅程

### 4.1 新用户发现 → 注册 → 学习路径

```
发现 (搜索引擎/SNS)
  ↓
访问首页 → 浏览模块卡片
  ↓
阅读免费文章 → 遇到每日一题
  ↓
注册账号 → 开始学习路径
  ↓
进阶 → 订阅会员
  ↓
完成路径 → 获得修了证书
  ↓
查看案件 → 投递简历
```

### 4.2 核心用户故事

#### US-001: 浏览首页
> 作为**访客**，我希望打开网站就能看到清晰的模块分类和最新内容，以便快速找到我感兴趣的 SAP 主题。

#### US-002: 模块浏览
> 作为**学习者**，我希望按 SAP 模块（FI/CO/MM 等）浏览文章，并能按难度（初/中/上級）筛选，以便找到适合我当前水平的文章。

#### US-003: 文章阅读
> 作为**学习者**，我希望阅读文章时有清晰的目录、友好的对话式讲解和代码示例，以便理解复杂的 SAP 概念。

#### US-004: 每日一题
> 作为**学习者**，我希望每天有一个 SAP 知识问答，答题后有详细解析和正解反馈，以便巩固所学知识。

#### US-005: 学习路径
> 作为**新人**，我希望有按角色设计的学习路径，按顺序学习，以便系统化掌握 SAP 知识。

#### US-006: 案件查找
> 作为**SAP 自由职业者**，我希望查看最新的 SAP 案件列表，按模块筛选，查看案件详情和薪资范围，以便找到合适的工作机会。

#### US-007: 案件投递
> 作为**SAP 顾问**，我希望从感兴趣的案件直接投递简历，填写个人信息并上传履历书，以便快速申请案件。

#### US-008: 视频学习
> 作为**学习者**，我希望观看 SAP 教学视频，包括操作演示和技术讲解，以便通过视觉方式学习。

#### US-009: AI 问答
> 作为**用户**，我希望向 AI 助手咨询 SAP 相关问题，获得即时解答，以便快速解决工作中遇到的问题。

#### US-010: 主题切换
> 作为**用户**，我希望切换网站配色主题和字体大小，调整动画强度，以便获得最佳的阅读体验。

### 5.3 功能详细描述

#### 5.3.1 首页 (F01)

#### 5.3.2 文章详情页 (F02)

**位置：** `/article.html`

**功能点：**
- 面包屑导航
- 文章标题、元数据（模块标签、难度标签、日期、阅读时长）
- 作者信息（头像、名称、角色）
- 文章封面图（模块风格化 SVG）
- 目录（Sticky TOC）
- 正文（H2/H3 标题、段落、列表、代码块、引用块）
- 熊猫对话框（パンダ先生 bubble）
- 学生对话框（たろうくん bubble）
- Callout Box（💡 提示 / ⚠ 警告）
- 文章反应（👍 ❤ 🎋 🙏 四按钮）
- 相关文章推荐（4 篇）
- 分享按钮（X / Facebook / LINE / Link）
- 侧边栏（TOC + 分享 + 下一篇）

#### 5.3.3 模块分类页 (F03)

**位置：** `/category.html`

**功能点：**
- 模块 Hero（图标 + 模块名 + 描述 + 统计）
- 熊猫思维 SVG 插图
- 侧边栏筛选器（Sticky）
  - 难度筛选（初級 / 中級 / 上級）
  - 主题筛选（基本概念 / マスタ設計 / トランザクション / 業務プロセス）
  - 学习路径推荐
- 文章网格（2 列卡片）
- 排序切换（新着順 / 人気順 / 難易度順）
- 分页器

#### 5.3.4 配色主题 (F05)

**3 套配色方案：**

| 主题 | 主色 | 风格 | 应用场景 |
|-----|------|-----|---------|
| 竹林（默认） | `#5A9D6E` 竹绿 | 自然、安心感 | 默认主题 |
| 温暖 | `#E07D3F` 柿橙 | 亲切、活力 | 适合学习场景 |
| 清新 | `#3B82F6` 蓝 | 清爽、专业 | 适合技术内容 |

通过 `data-theme` 属性 + CSS 变量切换，无需重新构建。

#### 5.3.5 每日一题 (F07)

**数据模型：**
- CPT: `daily_quiz`
- 字段：问题、4 个选项、正确答案、解析

**交互流程：**
1. 用户看到问题
2. 点击选项 → 即时反馈正/误
3. 显示解析（熊猫解説バブル）
4. 点击"次の問題" → 下一题
5. 扩展：登录后可记录正解率 + 连续正解数

#### 5.3.6 案件系统 (F09, F10, F13)

**数据模型：**
- CPT: `sap_case`（或自定义表）
- 字段：案件名、模块、単価、期間、勤務地、リモート、経験年数、必須スキル、歓迎スキル

**功能点：**
- 跑马灯展示最新案件
- 卡片列表（2 列网格）
- 模块筛选器（skill match）
- 案件详情模态框
- 投递表单（姓名、邮件、电话、希望单价、经验、技能选择、简历上传）

| 项目 | 要求 |
|-----|------|

- WCAG 可访问性：对比度、alt 属性、键盘导航
- `prefers-reduced-motion` 支持
- 动画强度控制（off/light/medium）

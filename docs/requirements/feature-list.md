## 2. P0 · 核心功能 (MVP)

> **优先级说明：** MVP 必须实现的功能，保证基础内容浏览体验。

## 8. 功能依赖关系图

```
Phase 1 (基盤)

       │

  ├── H-01 Header → H-02 Hero → H-03 模块卡片 → H-04 文章列表 → H-05 Footer

  └── H-06 响应式布局 (贯穿)
       │
Phase 3 (体验提升)
  ├── X-02~X-04 三套主题 → X-06 hover 动画
  ├── X-05 熊猫 SVG 动画
  └── H-07 SEO 基础
       │
Phase 4 (互动)
  ├── H-08 案件跑马灯 → J-01~J-04 案件系统
  ├── L-01 每日一题
  ├── L-02 学习路径 → H-09 路径展示
  └── H-10 Newsletter
       │
Phase 5 (增强)
  ├── U-01~U-03 用户系统 → L-03~L-05 学习进度 / 收藏 / 答题记录
  ├── C-07~C-09 排序 / 搜索 / 对话框

  └── I-01~I-04 Reaction / 分享 / 评论 / 保存

Phase 6 (高级)
  ├── AI-01~AI-03 AI 问答系统

  ├── L-06~L-07 证书 / 审批
  └── M-02~M-05 管理后台
```

## 9. 功能开发工作量估算

| 阶段 | 功能数 | 估算工时 | 包含 |
|------|--------|---------|------|
| Phase 1 | 3 | 1 天 | Docker / Nginx / 文章 CPT |
| Phase 2 | 10 | 3 天 | 首页布局 / 文章详情 / 分类页 / 响应式 |
| Phase 3 | 6 | 2 天 | 三套主题 / 熊猫动画 / SEO |
| Phase 4 | 8 | 3 天 | 案件系统 / Quiz / 学习路径 / Newsletter |
| Phase 5 | 14 | 5 天 | 用户系统 / 收藏 / 搜索 / 评论 |
| Phase 6 | 15 | 10 天 | AI 问答 / 积分 / 会员 / 管理后台 |
| 技术债 | 10 | 5 天 | 测试 / CI/CD / 安全 / 性能 |
| **总计** | **64** | **~29 天** | |

> **注：** 以上估算是单人纯开发时间，不含设计、测试、部署和修复时间。实际项目中建议乘以 1.5~2 倍的安全系数。

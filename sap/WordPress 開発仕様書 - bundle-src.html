01

整体设计概览

Design Overview

风格定位：治癒系卡通 + 知识杂志风混搭。以「パンダ先生（熊猫老师）」吉祥物为视觉锚点， 使用米色/竹绿主调降低 SAP 这类硬核技术内容的阅读门槛，同时保留杂志风的严谨排版（清晰层级、卡片+列表混排、明确的目录）。

核心设计原则

  • 降低门槛：用熊猫先生 vs 学生的对话框形式讲解枯燥的 SAP 概念。
  • 分类清晰：9 大 SAP 模块（FI/CO/MM/SD/PP/HR/ABAP/Basis/S/4）卡片式入口 + 三级难度标签（初/中/上級）。
  • 留白充足：容器最大 1280px，段落行高 1.7-1.95，文字大小 15-16px 起步。
  • 动效克制：仅熊猫眨眼/耳朵摇动 + Hero 浮动 + Hover 微抬升，无大块视差。
  • 3 套配色可切换：竹林（默认）/ 温暖讲堂 / 清新知識 — 通过 CSS 变量切换。

页面类型

  • ① Front Page（ホーム）：Hero → モジュール 9 卡 → 学習パス 3 件 → 新着＋人気 → 今日の一問 → YouTube → Newsletter
  • ② Category（モジュール一覧）：左侧難易度/トピック过滤器 + 右侧 2 列文章卡片网格 + ページネーション
  • ③ Single Post（記事詳細）：正文 + 右侧 Sticky TOC + 「パンダ先生 vs 学生」对话框 + 相关文章
アクセント 差別化要素：熊猫吉祥物贯穿所有页面（Logo、Hero、文章对话框、悬浮按钮、空状态、404）。 是整个品牌识别的核心 — 务必使用统一的 SVG 组件资产，避免不同页面用不同绘风。

06

动效与微交互

Motion & Micro-interactions

サイト全体の「呼吸感」「治癒系」雰囲気は、細やかな浮遊と表情の動きで作られている。 すべて純 CSS @keyframes + transform による実装で、JS は不要。 子テーマの style.css 末尾にまとめておく。

3 原则 不晕眩 / 不机械 / 不同期。 動かす量は ±6〜8px に抑え、必ず ease-in-out で。 複数要素を同時に動かすときは animation-delay や direction: reverse で位相をずらす — 全員が同じタイミングで動くと「団体行動」に見えて不自然になる。

A. 主体浮遊（Hero & 吹き出し）

@keyframes bobHero
Hero 中央：パンダ先生 / 太郎学生の上下浮遊。2 人で duration と delay を変えるのがコツ — 同期しないので自然なリズムが出る。
duration先生 5s / 学生 4.4s delay学生 +0.5s amplitudetranslateY(-6px) timingease-in-out infinite 適用.hero-panda-wrap / .hero-student-wrap 
@keyframes bobHero {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-6px); }
}
.hero-panda-wrap   { animation: bobHero 5s ease-in-out infinite; }
.hero-student-wrap { animation: bobHero 4.4s ease-in-out infinite;
                     animation-delay: 0.5s; }
@keyframes floatY
周囲の吹き出し（callout）と「?」マーク用。3 つの吹き出しを 0 / 1.2 / 2.4 秒ずらして順送りに浮かせることで、視線が自然に巡る。
SAPとは？ BAPI入門 MM 基礎
duration3〜4s（要素ごと差をつける） delay0 / 1.2 / 2.4 (s) amplitudetranslateY(-8px) directionq2 は reverse で逆位相 適用.callout / .qmark 
@keyframes floatY {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-8px); }
}
.callout    { animation: floatY 4s ease-in-out infinite; }
.callout.c2 { animation-delay: 1.2s; }
.callout.c3 { animation-delay: 2.4s; }

.qmark.q2   { animation: floatY 4s ease-in-out infinite reverse; }

B. SVG 内部の表情アニメ

キャラクターは <g class="..."> 単位で部位ごとに分け、CSS でアニメを掛ける。 transform-origin を耳の根本／手首に合わせるのが命 — ここが合っていないと「ねじれて」見えてしまう。

.eye-blink
4s / scaleY 1 → 0.05
.ear-wiggle
6s / ±6° (88-94% で)
.hand-wave
1.8s / -8° ↔ +14°
/* 部位 keyframes — 子テーマ style.css に追記 */
@keyframes blink     { 0%, 96%, 100% { transform: scaleY(1); } 98% { transform: scaleY(0.05); } }
@keyframes earWiggle { 0%, 100% { transform: rotate(0); } 92% { transform: rotate(-6deg); } 96% { transform: rotate(6deg); } }
@keyframes wave      { 0%, 100% { transform: rotate(-8deg); } 50% { transform: rotate(14deg); } }

.eye-blink  { transform-origin: center;        animation: blink 4s ease-in-out infinite; }
.ear-wiggle { transform-origin: bottom center; animation: earWiggle 6s ease-in-out infinite; }
.hand-wave  { transform-origin: bottom center; animation: wave 1.8s ease-in-out infinite; }

D. 強度コントロール & アクセシビリティ

ユーザーは右下のフローティングメニューから off / light / full を切り替えられる。 <html data-intensity="..."> 属性で制御し、選択は localStorage に保存する。

/* OFF：全アニメ停止 */
:root[data-intensity="off"] .eye-blink,
:root[data-intensity="off"] .ear-wiggle,
:root[data-intensity="off"] .hand-wave,
:root[data-intensity="off"] .hero-panda-wrap,
:root[data-intensity="off"] .callout,
:root[data-intensity="off"] .qmark        { animation: none !important; }

/* LIGHT：呼吸系は止め、瞬きだけ控えめに残す */
:root[data-intensity="light"] .eye-blink     { animation-duration: 6s; }
:root[data-intensity="light"] .ear-wiggle    { animation-duration: 10s; }
:root[data-intensity="light"] .hero-panda-wrap,
:root[data-intensity="light"] .callout       { animation: none !important; }
WCAG 必須 prefers-reduced-motion を必ず尊重する。 OS 側で「動きを減らす」設定をしているユーザーには、強度設定よりも優先して全停止させる。Lighthouse のアクセシビリティスコアにも影響する。 
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation: none !important;
    transition: none !important;
  }
}

E. WordPress 実装メモ

  • キーフレーム定義の場所：子テーマの style.css 末尾に集約する。Kadence の「Custom CSS」欄ではなく必ずファイルに置く（GZip が効くため）。
  • Hero パンダ：SVG は template-parts/hero-mascot.php として get_template_part() で読み込み。外側ラッパーに .hero-panda-wrap を付ければ自動で浮く。
  • 吹き出し：ACF Block 化し、ブロック設定で c1 / c2 / c3 を選べるようにする → delay が自動で決まる。
  • 強度切替 UI：footer.php に小さな JS（10 行程度）を仕込み、localStorage に保存。初回読み込み時は matchMedia('(prefers-reduced-motion: reduce)') を見て初期値を off に。
  • パフォーマンス：top / left ではなく transform のみを使うこと。GPU 合成が効き、Lighthouse の Avoid non-composited animations 警告も出ない。
  • パーツのクラス命名：先生・学生で同じクラスを共有させないこと（.ear-wiggle のように装飾的な接尾辞をつける）。delay 違いで管理できる。
